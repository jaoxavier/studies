# Leis fundamentais da Arquitetura de Software

## Lei 1 - Tudo é trade-off

- Não existem soluções perfeitas, apenas diferentes compensações.
- Toda decisão melhora algum aspecto às custas de outro. Se algo não parece ter trade-off, provabvelmente os trade-off não foram identificados
- Arquitetos devem estar sempre conscientes do que estão otimizando e que preço estão pagando

## Lei 2 - Por quê é mais importante que o Como

- Entender e registrar a motivação por trás das decisões arquiteturais é mais importante do que o detalhe de implementação de como o sistema foi montado
- O rationale das decisões guia futuras evoluções e permite a outros compreenderem a arquiteture
